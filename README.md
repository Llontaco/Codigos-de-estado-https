# Códigos de Estado para las Respuestas HTTP

## Descripción

Este proyecto contiene una referencia completa de los códigos de estado HTTP que se pueden recibir al realizar peticiones HTTP utilizando el método `get()` y otros métodos de solicitud.

Los códigos de estado HTTP son parte fundamental del protocolo HTTP y comunican el resultado de una solicitud HTTP. Están organizados en cinco categorías principales según el rango de números.


## Uso

Este documento sirve como referencia rápida para entender el significado de cada código de estado HTTP. Es útil para:

- **Desarrolladores web**: Para comprender las respuestas de los servidores.
- **Diseñadores de API**: Para asignar códigos de estado apropiados a diferentes respuestas.
- **Ingenieros de QA**: Para validar el comportamiento correcto de las aplicaciones web.
- **DevOps**: Para monitorear y diagnosticar problemas en servidores.

## Resumen Rápido

- **1xx**: Información - Solicitud recibida, continuando proceso
- **2xx**: Éxito - Acción completada exitosamente
- **3xx**: Redirección - Se requiere acción adicional
- **4xx**: Error del Cliente - Solicitud inválida o errónea
- **5xx**: Error del Servidor - El servidor no pudo procesar la solicitud
